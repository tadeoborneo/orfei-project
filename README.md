# Orfei Modernization Project

## 🎯 Overview
This project aims to design and implement a modern, scalable web application to replace a legacy internal management system ("Orfei"). The goal is to migrate outdated utilities into a robust digital ecosystem that enhances workflow efficiency, data integrity, and security.

## 🚀 Project Objectives
* **System Modernization:** Transitioning from a legacy environment to a high-performance web architecture.
* **Role-Based Access Control (RBAC):** Implementing a granular permission system to ensure data security and departmental isolation.
* **Operational Efficiency:** Automating manual tasks through a streamlined Backend-driven logic.
* **Professional Growth:** This repository serves as a showcase of clean code principles, professional documentation, and industry-standard Git workflow.

## 🛠 Tech Stack (Phase 1: Backend Core)
* **Language:** Python 3.12+
* **Framework:** FastAPI (High performance, asynchronous support)
* **Data Validation:** Pydantic V2
* **Database:** PostgreSQL
* **ORM:** SQLAlchemy
* **Authentication:** OAuth2 with JWT (Planned)

## 🏗 Architectural Approach
The project follows **Clean Architecture** principles to ensure the code is:
1.  **Independent of Frameworks:** The business logic doesn't depend on external libraries.
2.  **Testable:** Logic can be tested without the UI or Database.
3.  **Maintainable:** Clear separation of concerns between Models, Schemas, and Services.
